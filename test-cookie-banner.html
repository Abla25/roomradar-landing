Answers to Questions

1. How long are preferences maintained?

Answer: Preferences are saved in localStorage and persist for 30 days (GDPR compliance) until:

  • They automatically expire after 30 days
  • The user manually clears browser data
  • The banner version is updated (currently 1.0)
  • The user uses the "Clear Consent" button

2. Can preferences be modified?

Answer: Yes, in several ways:

  • "Customize" button in the main banner
  • "Update Cookie Preferences" button in the cookie policy page
  • JavaScript API: RoomRadarCookies.updateConsent()
  • "Show Preferences Modal" button on this test page

3. Is it in line with the site's style?

Answer: Yes, completely integrated:

  • Uses RoomRadar colors (#3B82F6 blue, #10B981 green)
  • Responsive and modern design
  • Smooth and consistent animations
  • Complete accessibility (WCAG compliant)

4. Doesn't reappear on other pages?

Answer: Correct, the banner:

  • Checks localStorage on every page load
  • Doesn't appear if consent has already been given
  • Works on all site pages
  • Is integrated in: index.html, how-it-works.html, faq.html, community.html, beta-signup.html, cookie-policy.html

5. How does it identify the same user?

Answer: The banner identifies the user through:

  • Browser localStorage: Data persists even after browser closure
  • Specific domain: Data is saved for roomradar.app
  • Unique key: 'roomradar-cookie-consent' identifies the consent
  • Timestamp: Checks when consent was given
  • Version: Verifies compatibility with updates

Note: If the user changes browser, device, or clears data, the banner will reappear.
